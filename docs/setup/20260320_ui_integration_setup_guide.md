# 통합 UI 및 기지 방어 시스템 설정 가이드 (2026-03-20 업데이트)

이 문서는 `UIManager.cs`와 `BaseManager.cs`를 사용하여 게임의 핵심 루프(기지 체력, 골드, 점수)를 구성하는 방법과 개발 과정에서 발생한 주요 문제들의 해결책을 담고 있습니다.

---

## 1. 시스템 구조 (Core Architecture)
- **BaseManager**: 기지의 체력과 충돌을 관리합니다. 체력이 변하면 이벤트를 쏩니다.
- **UIManager**: 기지의 신호를 받아 화면(Slider, Text)을 갱신합니다.
- **GameManager**: 골드와 점수를 관리하며 UIManager에게 화면 갱신을 요청합니다.

---

## 2. 필수 물리 설정 (Collision Setup)
적이 기지에 닿았을 때 체력이 깎이지 않는다면 다음 설정을 반드시 확인하세요.

### [1] 기지(Base) 설정
1. **Box Collider 2D** 추가 필수.
2. **Is Trigger** 체크 필수 (체크 안 하면 적이 기지에 부딪혀 튕겨나갑니다).
3. **Size**: X축을 화면 너비만큼(예: 20) 넓게 설정하여 '결승선'처럼 만듭니다.
4. **Z축 위치**: Transform의 Position Z값이 반드시 **0**이어야 합니다.

### [2] 적(Enemy) 설정
1. **Collider 2D** (Circle/Box)가 있어야 합니다.
2. **Rigidbody 2D**가 있어야 충돌이 감지됩니다.
   - **Gravity Scale**: **0** (중력 제거)
   - **Simulated**: 체크됨
3. **Tag**: 인스펙터 상단에서 태그를 반드시 **`Enemy`**로 설정하세요 (대소문자 주의).

---

## 3. UI(Slider) 시각적 연결법
슬라이더의 숫자(`Value`)는 변하는데 화면의 바가 움직이지 않을 때의 해결책입니다.

1. **Fill Rect 연결**: `BaseHPBar` 오브젝트 선택 -> `Slider` 컴포넌트의 **`Fill Rect`** 칸에 자식인 **`Fill`** 오브젝트를 드래그해서 넣으세요.
2. **색상 설정**: `Fill` 오브젝트의 `Image` 컴포넌트에서 색상을 초록색으로 설정하세요.
3. **인스펙터 확인**: 게임 실행 중 슬라이더를 클릭하고 `Value` 숫자가 실제로 줄어드는지 실시간으로 모니터링하세요.

## 4. 문제 해결 (Troubleshooting FAQ)

### Q: 콘솔에 "적 피격" 로그는 뜨는데 "기지 피격" 로그가 안 떠요.
- **A**: 그것은 플레이어가 쏜 총알이 적을 맞히고 있는 로그입니다. 기지 체력바와는 무관합니다. 적이 총알을 뚫고 화면 맨 아래까지 내려와서 기지 콜라이더에 닿아야 "기지 피격" 로그가 뜹니다.

### Q: 기지에 닿았다는 로그는 뜨는데 체력이 안 깎여요.
- **A**: `BaseManager`의 `OnTriggerEnter2D` 코드에서 태그 비교(`CompareTag("Enemy")`)를 확인하세요. 적 프리팹의 태그가 정확히 `Enemy`인지 다시 확인해야 합니다.

### Q: 게임 시작 시 체력바가 가득 차지 않아요.
- **A**: `UIManager`가 `BaseManager`보다 먼저 실행되어 초기값을 못 받은 경우일 수 있습니다. `UIManager` 인스펙터에서 `Hp Slider`의 초기 `Value`와 `Max Value`를 수동으로 10으로 맞춰두는 것도 좋은 방법입니다.
