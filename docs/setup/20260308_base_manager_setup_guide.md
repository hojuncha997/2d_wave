# 기지 관리 시스템 (BaseManager) 상세 설정 가이드 (2026-03-08)

이 문서는 고정형 타워 디펜스 슈팅의 핵심인 `BaseManager`와 5개의 `TowerSlot`을 유니티 에디터에서 설정하는 구체적인 방법을 단계별로 설명합니다.

---

## 1. 개요
`BaseManager`는 플레이어의 '기지' 역할을 수행하며, 전체 체력(HP) 관리, 타워 슬롯(TowerSlot) 총괄, 그리고 적 유닛이 방어선을 돌파했을 때의 데미지 처리를 담당합니다.

## 2. 유니티 에디터 설정 (Step-by-Step)

### [1단계] 기지(Base) 오브젝트 생성 및 설정
1. **오브젝트 생성**: `Hierarchy(하이어라키)` 창의 빈 공간에서 **마우스 우클릭 -> Create Empty**를 클릭합니다.
2. **이름 변경**: 새로 생긴 오브젝트의 이름을 **`Base`**로 변경합니다. (F2 키 활용)
3. **위치 설정**: `Inspector(인스펙터)` 창 맨 위의 **Transform** 컴포넌트 값을 다음과 같이 입력합니다.
    - **Position**: `X: 0, Y: -4.5, Z: 0`
4. **매니저 추가**: **Add Component** 버튼을 눌러 **`Base Manager`** 스크립트를 검색하여 추가합니다.
5. **충돌체 추가**: **Add Component** 버튼을 눌러 **`Box Collider 2D`**를 추가하고 아래 옵션을 설정합니다.
    - **Is Trigger**: 반드시 **체크(V)** 합니다. (물리적 밀려남 없이 충돌만 감지)
    - **Size**: `X: 12, Y: 1` (화면 가로 전체를 덮도록 설정)

### [2단계] 5개의 타워 슬롯(TowerSlot) 배치
1. **첫 번째 슬롯 생성**: Hierarchy 창에서 방금 만든 **`Base`** 오브젝트를 **마우스 우클릭 -> Create Empty**하여 자식(Child) 오브젝트를 만듭니다.
2. **이름 및 스크립트**: 이름을 **`Slot 1`**로 변경하고, **`Tower Slot`** 스크립트를 추가합니다.
3. **슬롯 위치**: `Slot 1`의 Transform Position을 `X: -2, Y: 0, Z: 0`으로 설정합니다.
4. **슬롯 복제**: `Slot 1`을 선택한 상태에서 **Ctrl+D (Cmd+D)**를 4번 눌러 `Slot 5`까지 만듭니다.
5. **간격 조정**: 각 슬롯의 **X 좌표**를 다음과 같이 수정하여 가로로 나란히 배치합니다.
    - **Slot 1**: `X: -2`
    - **Slot 2**: `X: -1`
    - **Slot 3**: `X: 0`
    - **Slot 4**: `X: 1`
    - **Slot 5**: `X: 2`

### [3단계] 기존 플레이어(Player) 정리
1. Hierarchy 창에서 기존 **`Player`** 오브젝트를 선택합니다.
2. 기존에 작동하던 이동/사격 로직이 제거되었으므로, 해당 오브젝트를 `Base` 위치로 옮겨 기지의 외형(Sprite)으로 재사용하거나, 필요 없다면 비활성화(Inspector 체크 해제)합니다.

---

## 3. 주요 기능 및 변수 (코드 레벨)
- **Singleton 패턴**: `BaseManager.Instance`를 통해 어디서든 접근 가능합니다.
- **체력 관리**: `_maxHp` (최대 체력), `_currentHp` (현재 체력) 변수를 통해 기지 파괴를 관리합니다.
- **슬롯 리스트**: `_towerSlots` 리스트는 실행 시 자식 오브젝트 중 `TowerSlot`이 붙은 것을 자동으로 감지하여 관리합니다.

## 4. 설정 확인 팁 (Troubleshooting)
- **Gizmos 활성화**: Scene 뷰 상단의 **`Gizmos`** 버튼이 켜져 있어야 타워 슬롯의 가이드라인(하얀색 사각형)이 보입니다.
- **적 유닛 설정**: 적이 기지를 그냥 통과한다면, 적 오브젝트의 **Tag**가 **`Enemy`**로 되어 있는지, 그리고 `Collider 2D`가 붙어 있는지 확인하세요.
- **데미지 확인**: 게임 실행 중 적이 `Base` 콜라이더에 닿았을 때 `Console` 창에 "기지 피격!" 로그가 찍히는지 확인합니다.

## 5. 시스템 간 연동
- **Enemy -> BaseManager**: 적 유닛이 `Base`의 트리거와 충돌 시 `TakeDamage(1)`를 호출합니다.
- **BaseManager -> GameManager**: 기지 체력이 0이 되면 `GameManager.Instance.EndGame()`을 호출하여 게임 오버 처리를 합니다.
