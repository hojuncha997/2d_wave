# 골드 및 타워 구매 시스템 설정 가이드 (2026-03-14)

이 문서는 게임 내 경제 시스템(골드)의 구현 방식과, 이를 사용하여 타워를 전략적으로 배치하는 시스템에 대해 설명합니다.

---

## 1. 시스템 개요
본 시스템은 플레이어가 적을 처치하여 자원을 획득하고, 이를 소비하여 방어 타워를 건설하는 핵심 게임 루프를 형성합니다.

- **획득**: 적(`Enemy`)이 파괴될 때 일정량의 골드를 지급합니다.
- **보유**: `GameManager`가 전체 골드량을 관리하며 UI에 표시합니다.
- **소비**: 빈 `TowerSlot`을 클릭하면 골드를 차감하고 타워를 생성합니다.

## 2. 주요 컴포넌트 상세

### [1] GameManager (자원 관리자)
- **역할**: 게임 내 유일한 골드 저장소 및 UI 업데이트 담당.
- **주요 변수**:
    - `Gold`: 현재 보유량 (기본값: 100G)
    - `_goldText`: 골드 표시용 TextMeshProUGUI 연결 필드.
- **주요 메서드**:
    - `AddGold(int amount)`: 골드 증가.
    - `UseGold(int amount)`: 골드 감소 (잔액 부족 시 `false` 반환).

### [2] Enemy (보상 지급자)
- **역할**: 사망 시 플레이어에게 전리품(골드)을 제공.
- **설정 항목**:
    - `Gold Reward`: 처치 시 지급할 골드 양 (기본값: 10G).

### [3] TowerSlot (구매 상호작용)
- **역할**: 클릭 이벤트를 감지하여 타워 건설 로직 실행.
- **설정 항목**:
    - `Tower Price`: 타워 건설 비용 (기본값: 50G).
    - `Is Occupied`: 이미 타워가 있는지 여부 체크.
- **동작**: `OnMouseDown` 호출 시 골드 잔액을 확인 후 `DeployTower()` 실행.

---

## 3. 유니티 에디터 필수 설정 (Setup)

### [1단계] 골드 UI 생성 및 연결
1. **Canvas** 아래에 새로운 **TextMeshPro - Text** 오브젝트를 생성합니다. (기존 Score UI 복제 권장)
2. 이름을 **`GoldText`**로 변경하고 화면 상단 적절한 위치에 배치합니다.
3. `Hierarchy`에서 **`GameManager`** 오브젝트를 선택합니다.
4. `Inspector`의 **`Gold Text`** 슬롯에 방금 만든 `GoldText` 오브젝트를 드래그하여 연결합니다.

### [2단계] 타워 슬롯 클릭 감지 설정
1. 각 **`Slot 1 ~ 5`** 오브젝트에 **`Box Collider 2D`**가 붙어 있는지 확인합니다.
2. 콜라이더의 **Size**가 `TowerSlot` 스크립트의 Gizmo 사각형 크기(기본 1x1)와 일치하는지 확인합니다.
3. **Is Trigger** 옵션은 체크되어 있어도 무방합니다 (`OnMouseDown` 작동 가능).

---

## 4. 밸런스 조절 팁
- **초기 자금**: `GameManager`의 `Gold` 초기값을 수정하여 시작 시 타워 개수를 조절할 수 있습니다.
- **난이도 조절**: `Enemy` 프리팹의 `Gold Reward`를 낮추거나 `TowerSlot`의 `Tower Price`를 높여 게임의 긴장감을 높일 수 있습니다.

## 5. 트러블슈팅 (Q&A)
- **Q: 클릭해도 타워가 안 생겨요!**
    - A1: 슬롯에 `Box Collider 2D`가 있는지 확인하세요.
    - A2: `GameManager`에 `Gold Text`가 연결되지 않아 NullReference 에러가 나는지 콘솔을 확인하세요.
    - A3: 골드가 부족한지 콘솔 로그를 확인하세요. (필요: 50G)
- **Q: 타워 가격을 다르게 하고 싶어요.**
    - A: 각 슬롯(`Slot 1~5`) 인스펙터에서 `Tower Price`를 개별적으로 수정하거나, 프리팹에서 일괄 수정할 수 있습니다.
