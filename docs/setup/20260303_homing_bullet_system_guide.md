# 자동 추적(Homing) 총알 시스템 가이드 (2026-03-03)

이 문서는 타워가 발사한 총알이 적을 자동으로 추적하여 명중률을 높이는 시스템의 구현 원리와 설정 방법을 설명합니다.

## 1. 자동 추적(Homing)의 원리
기존 총알은 생성된 후 무조건 `Vector3.up`(위쪽)으로만 이동했습니다. 
자동 추적 총알은 다음 단계를 거칩니다:
1. **타겟 지정**: 타워가 총알을 생성할 때, 현재 조준 중인 적(`Enemy`) 정보를 총알에게 넘겨줍니다.
2. **방향 계산**: 매 프레임마다 총알은 적의 현재 위치와 자신의 위치 차이를 계산합니다.
3. **추적 이동**: 총알이 적의 위치를 향해 방향을 꺾으며 계속 전진합니다.

## 2. 코드 구조 변경 사항
### A. `Bullet.cs` (수정 예정)
- `SetTarget(Transform target)` 메서드를 추가하여 추적 대상을 저장합니다.
- `Update()` 메서드에서 단순히 위로 가는 대신, `_target` 방향으로 이동하게 합니다.
- 만약 추적하던 적이 도중에 죽으면, 총알도 스스로 파괴되거나 마지막 방향으로 직진하도록 처리합니다.

### B. `Tower.cs` (수정 예정)
- `Shoot()` 메서드에서 총알을 생성한 직후, `bullet.SetTarget(_target)`을 호출하여 정보를 전달합니다.

## 3. 유니티 에디터 설정 주의사항
- **속도 조절**: 추적 총알은 너무 빠르면 회전 반경이 커져서 적을 지나칠 수 있습니다. `Speed` 값을 적절히 조절하며 테스트해야 합니다.
- **충돌체(Collider)**: 추적 기능이 있더라도 `Box Collider 2D`와 `Is Trigger` 설정은 그대로 유지되어야 적과 부딪혔을 때 파괴됩니다.

## 4. 기대 효과
- **명중률 100%**: 적이 좌우로 움직이거나 속도가 빨라져도 타워가 끝까지 쫓아가서 맞춥니다.
- **시각적 재미**: 총알들이 곡선을 그리며 적에게 꽂히는 모습이 더 역동적으로 보입니다.
