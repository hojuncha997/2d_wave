# 타 기기에서 프로젝트 클론 및 세팅 가이드 (2026-03-03)

이 문서는 다른 컴퓨터에서 GitHub 리포지토리를 클론(`git clone`)하여 이 프로젝트를 처음 열 때 필요한 설정 과정을 설명합니다.

## 1. 프로젝트 가져오기
1. **클론**: 터미널에서 다음 명령어를 실행하여 리포지토리를 내려받습니다.
   ```bash
   git clone https://github.com/hojuncha997/2d_wave.git
   ```
2. **Unity Hub 실행**: `Add` 버튼을 눌러 클론한 폴더를 프로젝트 목록에 추가합니다.
3. **버전 확인**: 이 프로젝트는 Unity **2022.3.x** (또는 그 이상) 버전을 권장합니다. (사용 중인 버전이 다르면 업그레이드 여부를 묻는 팝업이 뜰 수 있습니다.)

## 2. 첫 실행 시 주의사항 (중요)
1. **긴 로딩 시간**: 처음 프로젝트를 열면 `Library` 폴더를 생성하는 과정에서 시간이 수 분 정도 소요될 수 있습니다. (정상이니 기다려 주세요.)
2. **빈 화면이 보일 때**: 유니티가 처음 열리면 빈 씬이 나타날 수 있습니다.
   - **해결**: 프로젝트 창에서 `Assets/Scenes/SampleScene.unity` 파일을 **더블클릭** 하세요. 우리가 작업하던 모든 물체(Player, Enemy, HUD 등)가 나타납니다.

## 3. TextMeshPro(TMP) 리소스 확인
우리는 이미 TMP 리소스를 커밋했으므로 별도로 다운로드할 필요는 없으나, 유니티 엔진이 내부적으로 이를 다시 확인하는 팝업을 띄울 수 있습니다.
- 만약 "TMP Importer" 창이 다시 뜬다면, **"Import TMP Essentials"**를 한 번 더 눌러주어 유니티 엔진에 현재 라이브러리가 존재함을 확인시켜 주세요.

## 4. 인스펙터 칸이 비어 있을 때 (Reference Missing)
드문 경우지만, 유니티 엔진이 오브젝트 간의 연결(드래그 앤 드롭한 것들)을 잃어버릴 때가 있습니다. 만약 게임 실행 시 에러가 난다면 다음을 확인하세요.
1. **GameManager**: `Score Text`, `GameOver UI` 등이 제대로 연결되어 있는지 확인.
2. **Player**: `Bullet Prefab`이 제대로 연결되어 있는지 확인.
3. **EnemySpawner**: `Enemy Prefab`이 제대로 연결되어 있는지 확인.

## 5. 결론
이 프로젝트는 모든 핵심 자산(코드, 프리팹, 씬, 라이브러리)이 Git에 포함되어 있으므로, 클론 후 **SampleScene을 여는 것**만으로도 즉시 개발을 재개할 수 있습니다.
