# 웨이브 시스템 설정 가이드 (2026-03-16)

이 문서는 `2d_wave` 프로젝트의 적 생성 방식을 시간 기반 무한 생성에서 **데이터 기반 웨이브 시스템**으로 전환하기 위한 설계 가이드입니다.

---

## 1. 시스템 개요
- **목적**: 정해진 적의 종류와 마리수를 웨이브 단위로 소환하여 플레이어에게 명확한 성취감을 제공함.
- **핵심 흐름**:
    1. 웨이브 데이터(`WaveData`) 로드.
    2. 데이터에 정의된 만큼 적 소환.
    3. 소환된 적이 모두 처치되면 일정 시간 대기(Intermission).
    4. 다음 웨이브 시작.

---

## 2. 주요 구성 요소

### [1] WaveData (ScriptableObject)
유니티 에디터에서 웨이브 설정을 시각적으로 관리하기 위한 데이터 형식입니다.
- `int WaveNumber`: 웨이브 번호.
- `List<EnemyBatch> Batches`: 해당 웨이브에서 소환할 적의 묶음 정보.
    - `GameObject EnemyPrefab`: 적 종류.
    - `int Count`: 소환할 마리수.
    - `float SpawnInterval`: 이 배치 내에서의 소환 간격.
- `float WaitTimeAfterWave`: 웨이브 종료 후 다음 웨이브까지의 대기 시간.

### [2] EnemySpawner (수정)
기존의 `Update` 로직을 제거하고, `WaveManager`의 명령에 따라 적을 순차적으로 소환하는 기능을 수행합니다.
- `SpawnWave(WaveData data)`: 전달받은 데이터를 바탕으로 코루틴(Coroutine) 등을 활용해 적을 소환함.

### [3] WaveManager (신규)
전체적인 게임 진행의 '흐름'을 제어하는 핵심 매니저입니다.
- 현재 웨이브 번호 관리.
- 화면에 "Wave 1 Start!"와 같은 UI 메시지 표시 트리거.
- 모든 적이 죽었는지 확인하여 다음 웨이브로 넘김.
