# 씬 관리 최적화 및 프리팹 중심 개발 전략 (2026-03-17)

이 문서는 프로젝트가 커짐에 따라 씬(.unity) 파일이 비대해지는 것을 방지하고, 유지보수 및 AI 협업 효율성을 높이기 위한 전략을 설명합니다.

---

## 1. 왜 씬 파일을 가볍게 유지해야 하는가?
- **협업 충돌 방지**: 씬 파일은 텍스트(YAML) 기반이라 여러 사람이 동시에 수정하면 충돌을 해결하기 매우 어렵습니다.
- **로딩 속도 및 성능**: 씬이 무거워지면 에디터 로딩과 런타임 초기화 속도가 느려집니다.
- **AI 협업 효율**: AI가 수천 줄의 씬 파일을 분석하는 것보다, 독립된 프리팹 파일을 분석하는 것이 훨씬 빠르고 정확합니다.

---

## 2. 핵심 관리 전략

### [1] 프리팹(Prefab) 우선 원칙
- **정의**: 모든 게임 오브젝트(타워, 적, 마커, UI 패널 등)는 반드시 프리팹으로 제작하여 보관합니다.
- **효과**: 씬 파일에는 프리팹의 위치와 기본 정보만 저장되고, 상세 로직과 수치는 외부 프리팹 파일(.prefab)에 분산 저장됩니다.

### [2] 데이터와 로직의 분리 (ScriptableObject)
- **정의**: 웨이브 설정, 유닛 능력치 등의 수치는 씬 안의 오브젝트에 직접 적지 않고, `ScriptableObject`(.asset)에 저장합니다.
- **효과**: 씬을 열지 않고도 게임 밸런스를 조절할 수 있으며, 씬 파일 크기를 획기적으로 줄입니다.

### [3] 멀티 씬 에디팅 (Multi-Scene Editing)
- **방법**: 하나의 게임을 기능별로 여러 씬으로 쪼개서 관리합니다. (예: `SystemScene`, `UIScene`, `LevelScene`)
- **장점**: 특정 부분(예: UI)만 수정할 때 해당 씬만 읽고 작업하면 되므로 효율적입니다.

---

## 3. AI(Gemini CLI)와 협업 시 팁

씬이 복잡해졌을 때 AI에게 다음과 같이 요청하면 분석 정확도가 올라갑니다.

- **"씬 전체를 읽지 말고, XXX 프리팹 파일만 읽어서 분석해줘."**
- **"특정 오브젝트의 컴포넌트 설정이 이상한데, .prefab 파일의 수치를 확인해줘."**
- **"밸런스 데이터인 WaveData.asset 파일을 읽어서 오류가 없는지 봐줘."**

---

## 4. 실천 지침
1. 새로운 기능을 만들 때는 먼저 **빈 오브젝트에서 테스트 후 즉시 프리팹화** 하세요.
2. 씬에 직접 배치된 오브젝트(Scene Object)의 수를 최소화하고, 가급적 **런타임 소환(Instantiate)** 방식을 사용하세요.
3. 씬 파일이 1MB를 넘어가기 시작하면, 멀티 씬이나 프리팹 구조를 재점검하세요.
