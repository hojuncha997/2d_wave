# 기획 피벗: 고정형 타워 디펜스 슈팅 (2026-03-03)

이 문서는 기존의 이동형 슈팅 게임에서 고정형 타워 디펜스 스타일로 게임 디자인을 변경하는 이유와 상세 기획을 담고 있습니다.

## 1. 변경 배경: 모바일 UX 최적화
- **문제점**: 손가락을 화면에 붙여 계속 드래그하는 방식은 사용자 피로도가 높고 화면을 가림.
- **해결책**: 플레이어의 위치를 고정하고, **'선택과 배치'** 위주의 전략적 재미에 집중함.

## 2. 핵심 게임 메커니즘 (변경사항)
- **고정형 베이스**: 플레이어(또는 기지)는 화면 하단에 고정됨.
- **5개의 타워 슬롯**: 기지에는 최대 5개의 타워를 배치할 수 있는 슬롯이 존재함.
- **타워 다양성**:
    - 타워마다 공격력, 발사 속도, 사거리(Range)가 다름.
    - 특수 능력(예: 범위 공격, 둔화, 관통 등)을 가진 타워 도입 가능.
- **적 유닛**: 적들은 화면 위에서 아래로 내려오며, 고정된 기지나 타워 라인을 파괴하려 함.

## 3. 기술적 변경 로직 (To-Do)
- [ ] **PlayerController 폐기/수정**: 좌우 이동 로직을 제거하고 '베이스(기지)' 관리자로 변경.
- [ ] **Tower 클래스 생성**: 개별 타워의 공격 로직과 사거리 감지 기능을 담당.
- [ ] **TowerManager 생성**: 5개의 슬롯 관리 및 타워 배치 시스템.
- [ ] **사거리 시각화**: 타워를 선택하거나 배치할 때 사거리를 원형으로 표시하는 기능.

## 4. 기대 효과
- **전략적 재미**: 어떤 타워 5개를 조합하느냐에 따라 공략 방법이 달라짐.
- **낮은 피로도**: 한 번 배치하면 자동으로 사격하거나 필요할 때만 타겟팅을 변경하면 됨.
- **확장성**: 추후 새로운 타워 종류를 추가하기 매우 쉬운 구조가 됨.
